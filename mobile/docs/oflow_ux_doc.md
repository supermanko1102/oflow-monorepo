# OFlow 行動應用 UX 文件

## 一、使用者旅程概覽

1. **發現需求**：商家收到大量 LINE 訂單，感到整理困難。
2. **註冊登入**：選擇 LINE/Apple 登入，授權 OFlow 存取必要資訊。
3. **導覽與設定**：了解介面與提醒功能，設定團隊與通知偏好。
4. **日常使用**
   - 自動同步 LINE 訂單 → 進入訂單列表 → 檢視詳情 → 更新狀態。
   - 定時檢視提醒頁 → 跟進即將到期訂單 → 標記完成或延後。
5. **後續維護**：調整商品目錄、邀請新成員、檢視營運趨勢。

## 二、核心體驗原則

- **降低認知負擔**：將任務拆分為「檢視」「處理」「追蹤」三個步驟。
- **建立信任**：清楚標示 AI 解析來源，提供人工覆核與修正。
- **即時可見**：強調待辦提醒與狀態變化，確保關鍵資訊不被忽略。
- **協作友好**：明確顯示負責人與變更紀錄，支援團隊溝通。

## 三、關鍵情境與流程

### 情境 A：早晨整理訂單

1. 登入後 Dashboard 顯示今日待處理數量。
2. 使用者點入訂單列表，切換到「待處理」篩選。
3. 逐筆打開訂單詳情，確認 AI 解析內容。
4. 若需補充資訊，於備註欄新增；必要時直接開啟 LINE 聯絡客戶。
5. 處理完畢後將訂單標記完成或安排提醒。

### 情境 B：追蹤提醒

1. 中午時段進入提醒頁，查看「今日」清單。
2. 對於已處理的訂單向右滑標記完成。
3. 若需延後，向左滑選擇「延後一天/自訂時間」。
4. 完成後提醒清單即時更新，並顯示成功 Toast。

### 情境 C：新增團隊成員

1. 店主在設定頁開啟「團隊與成員」。
2. 生成邀請碼，透過 LINE 傳送給新成員。
3. 新成員登入後輸入邀請碼加入團隊。
4. 店主檢視成員列表並設定角色權限。

## 四、資訊架構（IA）

- **層級 1**：主導航（Dashboard、訂單、提醒、設定）
- **層級 2**
  - 訂單：列表、詳情、搜尋
  - 提醒：提醒列表、延後、完成歷史
  - 設定：帳號、通知、團隊、模式、關於
- **輔助導航**
  - 底部 Sheet：快速建立訂單、延後提醒
  - 浮動按鈕（未來延伸）：掃描 LINE 對話

## 五、互動策略

- **反饋機制**
  - 行動成功：顯示帶圖示的 Toast（例如「提醒已完成」）。
  - 錯誤：顯示對話框，提供重試與聯絡支援選項。
  - Loading：骨架屏或按鈕 Loading 動畫，避免空白。
- **手勢支援**
  - 列表滑動操作（完成/延後）、長按加入快速標籤（未來規劃）。
- **提醒邏輯**
  - 系統在取貨前自動推播，同步更新 App 內提醒。
  - 使用者可自訂時間或重複頻率。

## 六、痛點與解法

| 痛點               | 使用者感受         | UX 解法                            |
| ------------------ | ------------------ | ---------------------------------- |
| 大量訂單難以整理   | 容易遺漏、資訊混亂 | 使用分類篩選 + 一目了然的卡片設計  |
| 取貨提醒無一致流程 | 常忘記追蹤         | 智慧提醒 + 推播 + 滑動操作快速標記 |
| 團隊協作缺乏透明度 | 不知道誰負責       | 顯示負責人頭像與狀態流             |
| AI 解析可信度不足  | 擔心錯誤導致災難   | 提供 AI 原文對照、人工覆核流程     |

## 七、量化檢驗指標

- NPS ≥ 40，特別關注「節省整理時間」與「提醒可靠」兩項回饋。
- 每日活躍使用者（DAU）至少 70% 進入提醒頁面。
- 90% 的訂單在建立後 24 小時內被標記為「處理中」或「完成」。
- 介面操作平均任務完成時間（如標記提醒完成） < 4 秒。

## 八、使用性測試計畫

1. **對象**：5 位 LINE 商家（甜點、飲料、花店等）。
2. **任務**
   - Task 1：登入後找到待辦訂單並更新狀態。
   - Task 2：從提醒頁延後一筆提醒到明天。
   - Task 3：邀請店員加入並設定為「管理者」。
3. **指標**
   - 任務成功率 ≥ 90%。
   - 平均任務完成時間 < 2 分鐘。
   - SUS（System Usability Scale）≥ 75。
4. **後續行動**
   - 收集錄影與口語回饋，整理改善清單。

## 九、未來體驗延伸

- 導入教學導覽（Onboarding Coach Marks）。
- 建立關鍵情境的推播策略（例如大量訂單尖峰提醒）。
- 導入 AI 建議回覆模板，縮短與客戶溝通時間。
- 提供多帳號快速切換（顧問角色需求）。
