# OFlow 行動應用 UI 文件

## 一、設計原則

- **一致性**：各頁面採用相同的排版網格、間距與色彩系統，維持品牌識別。
- **可讀性**：善用字體階層（Title、Headline、Body）、高對比配色確保資訊清晰。
- **可操作性**：所有互動元件遵循 Material Design 與 NativeWind 標準，提供 44px 以上觸控區域。
- **狀態清楚**：針對載入中、成功、失敗、空狀態提供獨立視覺。

## 二、色彩與字體

- **主色**：`#00B900`（LINE 綠）
- **輔色**：`#0F172A`（深藍灰）、`#F97316`（提醒橘）、`#6366F1`（半自動紫）
- **狀態色**
  - 成功：`#22C55E`
  - 警示：`#FACC15`
  - 錯誤：`#EF4444`
- **字體**
  - iOS：PingFang TC
  - Android：Noto Sans TC
  - 字級：Title 24、Headline 20、Body 16、Caption 14

## 三、主要畫面規格

### 1. 登入頁 (`app/(auth)/index.tsx`)

- **結構**
  - 頂部品牌 Logo 與標語。
  - 中段功能亮點卡片（自動讀取、智慧提醒、模式切換）。
  - 底部登入按鈕區：LINE、Apple。
- **互動**
  - 登入按鈕採用全寬設計，提供 loading 與 disabled 狀態。
  - 深色模式下背景改為 `#111827`，按鈕保持品牌色。
- **錯誤處理**
  - 登入失敗顯示 Snackbar：「登入失敗，請稍後再試」。

### 2. 首頁 Dashboard (`app/(tabs)/index.tsx`)

- **內容**
  - 今日提醒摘要（未完成數量）。
  - 本週訂單概況：折線圖或柱狀圖簡化呈現。
  - 快捷操作卡片（建立訂單、掃描 LINE 訊息）。
- **元件**
  - 使用 `Card` 元件包裝，提供陰影與圓角。
  - 圖表使用簡化版（占位圖，待後續資料統合）。

### 3. 訂單列表 (`app/(tabs)/orders.tsx`)

- **篩選列**
  - Tab：全部 / 待處理 / 已完成 / 已取消。
  - 次級篩選：`Select` 切換全自動/半自動。
- **列表項（OrderCard）**
  - Header：客戶名稱 + 狀態徽章（顏色依類型）。
  - Body：訂單摘要（商品、金額），AI 模式標籤。
  - Footer：取貨日期時間、負責人頭像。
- **空狀態**
  - 顯示插圖、文字「目前沒有訂單」，提供「建立新訂單」按鈕。

### 4. 訂單詳情 (`app/order/[id].tsx`)

- **資訊區塊**
  1. 訂單基本資料（客戶、模式、金額、狀態切換器）。
  2. 明細列表（商品、數量、單價、小計）。
  3. LINE 對話區（氣泡式，標示 AI 解析來源）。
  4. 動作列：標記完成、聯絡客戶、加到提醒。
- **視覺**
  - 使用分隔線將區塊分開，背景採用 `#F8FAFC`。
  - 操作按鈕使用 `Button`，提供協助 Tooltip 說明。

### 5. 提醒頁 (`app/(tabs)/reminders.tsx`)

- **列表結構**
  - SectionList：今日 / 3 日內 / 7 日內。
  - 每筆提醒顯示：客戶、剩餘時間、訂單摘要、優先度。
- **互動**
  - 向右滑：標記完成（綠色背景）。
  - 向左滑：延後提醒，彈出底部 Sheet 選擇時間。
- **空狀態**
  - 顯示「目前沒有未完成提醒」，提供「建立提醒設定」連結。

### 6. 設定頁 (`app/(tabs)/settings.tsx`)

- **模組**
  - 帳號資訊：顯示登入提供者、帳號、登出按鈕。
  - 團隊與成員：成員列表、角色徽章、邀請碼。
  - 通知設定：切換開關與時間選擇器。
  - 模式設定：全自動 / 半自動，提供文字說明卡片。
- **視覺**
  - 使用 List.Item 圖標與標題，右側放置控制元件。

## 四、元件規格

### Button

- Variant：主按鈕（填色）、次按鈕（描邊）、純文字。
- 狀態：預設、Disabled、Loading。
- 當 Loading 時顯示 ActivityIndicator，文字透明度降低。

### StatusBadge

- 全自動：背景 `#0EA5E9` / 文字白。
- 半自動：背景 `#6366F1` / 文字白。
- 待處理：背景 `#F97316` / 文字白。
- 已完成：背景 `#22C55E` / 文字白。
- 已取消：背景 `#EF4444` / 文字白。

### Toast

- 類型：成功、警示、錯誤、資訊。
- 顯示位置：頁面底部置中，最大寬度 320px。
- 自動消失 3 秒，可手動關閉。

### Form Inputs

- 採用 `TextInput` 搭配 `HelperText` 顯示驗證訊息。
- 必填欄位標示 `*`（紅色），錯誤訊息使用紅色 Caption。

## 五、狀態與錯誤

- **Loading**：使用 Skeleton 或 ActivityIndicator。
- **Empty**：提供插圖 + 文案 + 行動按鈕。
- **Error**：顯示錯誤訊息與重試按鈕，並記錄 log。
- **Offline**（未來規劃）：顯示橘色橫幅「目前為離線模式，將於恢復網路後同步」。

## 六、無障礙與在地化

- 所有互動元件提供語音標籤（label，對應繁體中文）。
- 保持文字最小 14pt，行距 1.4。
- 支援動態字體（Dynamic Type），按比例調整間距。
- 深色模式下調整背景與文字對比（WCAG AA）。

## 七、資源需求

- UI Kit：Figma 連結（待建立），包含色彩、字體、元件、圖示。
- 插圖來源：使用 Lottie 或 SVG，需提供 light/dark 兩版。
- Icon 套件：`@expo/vector-icons` 與 `expo-symbols`。
